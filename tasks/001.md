# Task 001: Animation editor in SpriteEditor

## Goal

Add a lightweight animation-preview feature to `src/editor/SpriteEditor.cs`. No new `IEditor` tab — this lives inside the existing sprite editor, reusing its current tools/palette/flags UI.

An "animation" is just an ordered sequence of up to 8 sprite indices (frame slots 1-8), assigned from the sprite currently selected on the canvas (`sprNmbr`). It previews looping/scrubbing through those frames at a configurable zoom and speed.

## New UI layout

All new rects fit inside the free `x:0-99, y:9-86` region, laid out to mirror the existing sprite-canvas/palette relationship (frame strip plays the role of a mini "timeline" to the left of the preview, controls stacked between it and the preview box):

| Element | Rect (x, y, w, h) | Notes |
|---|---|---|
| `animFrameSlots[8]` | x=0, y=15+i*8, w=8, h=8 (i=0..7) | Vertical stack, 64px tall, aligned with preview box below. Visual only, not clickable — driven by keys 1-8. |
| `animZoomBtn` | 10, 15, 22, 8 | Click cycles `Zooms = {1,2,4,8}` (reuse the same array already defined for the sprite canvas). Label: `x1`/`x2`/`x4`/`x8`. |
| `animSpeedBtn` | 10, 25, 22, 8 | Click cycles `AnimSpeeds = {2, 4, 8, 12}` (fps). Label: speed value as text. |
| `animLoopModeBtn` | 10, 35, 22, 8 | Click cycles `Pause → Forward → Reverse → PingPong → Pause`. Label: `PA`/`FW`/`RV`/`PP`. |
| `animPreviewArea` | 32, 15, 64, 64 | Same size/style as `sprcnvsarea`: black rectfill background (`rectfill` at x-1,y-1..x+w,y+h), draws the current playback frame via `_api.spr(...)` scaled by `Zooms[AnimSclIdx]`, same pattern as `CurrentCanvasRegion()`/`CnvScale`. |

Button visuals for `animZoomBtn` / `animSpeedBtn` / `animLoopModeBtn` follow the existing text-label pattern used for `sprNmbrLabelArea` (`rectfill` with `Constants.Colors.LightGray` background, `_api.print(...)` with `Constants.Colors.Indigo` text) rather than `Button`/icons, since no icon indices are confirmed free/appropriate for text-length toggle states. Click detection: `rect.Contains(mouse.x, mouse.y) && _api.mouselp()`, same as other click handling in `Update`.

`animFrameSlots[i]` rendering: if `AnimFrames[i] != -1`, draw the assigned sprite thumbnail via `_api.spr(AnimFrames[i], x, y, 1, 1, 1)` (8x8, scale 1 fits exactly); if empty, draw a dim placeholder (reuse `DrawEmptyWorkspacePattern` at a small scale, or a flat dark rect). While playing, draw a 1px white border around the slot that is currently the active/displayed frame.

## New state fields

```csharp
private int[] AnimFrames = { -1, -1, -1, -1, -1, -1, -1, -1 }; // -1 = empty slot, else sprite index
private int AnimSclIdx = 0;            // index into Zooms, independent of SprSclIdx
private int[] AnimSpeeds = { 2, 4, 8, 12 }; // frames per second
private int AnimSpeedIdx = 0;
private enum LoopMode { Pause, Forward, Reverse, PingPong }
private LoopMode animLoopMode = LoopMode.Pause;
private int animCurrentFrame = 0;      // index into AnimFrames (0-7) currently displayed
private int animPingPongDir = 1;       // +1 or -1, only used in PingPong mode
private float animElapsed = 0f;        // time accumulator for playback
```

## Keyboard input (in `Update`, alongside existing key handling)

- **Keys 1-8** (`Keys.D1`..`Keys.D8`), `JustPressed`: toggle frame slot `i = key - 1`.
  - If `AnimFrames[i] == -1`: set `AnimFrames[i] = sprNmbr` (current selected sprite).
  - Else: set `AnimFrames[i] = -1` (clear the slot).
  - Only handle this when no modifier keys are pressed, and skip if it collides with any existing shortcut (none currently use plain number keys).
- **Key 9** (`Keys.D9`), `JustPressed`: clear all — set every `AnimFrames[i] = -1`.

## Playback behavior

Let `filled` = indices `i` where `AnimFrames[i] != -1`. If `filled` is empty, nothing to preview (draw the preview box empty/placeholder). Otherwise let `first`/`last` be the min/max index in `filled` — playback only ranges over `[first, last]`, skipping any empty slots inside that range (don't display/count them as a step).

- **Pause**: don't advance `animElapsed`/`animCurrentFrame`. Preview shows `AnimFrames[first]` (or the last frame actually displayed before pausing — no auto-reset needed).
- **Forward**: advance through filled frames in ascending index order; after `last`, wrap to `first`.
- **Reverse**: advance through filled frames in descending index order; after `first`, wrap to `last`.
- **PingPong**: advance forward from `first` to `last`, then reverse back down to `first`, repeating (don't double-display the endpoints on direction change).

Advance timing: accumulate `elapsedSeconds` into `animElapsed`; when it exceeds `1f / AnimSpeeds[AnimSpeedIdx]`, step to the next filled frame per the current mode and reset the accumulator (subtract the interval, not zero, to avoid drift).

Draw the resulting `AnimFrames[animCurrentFrame]` sprite in `animPreviewArea`, scaled by `Zooms[AnimSclIdx]`, clipped to the box (mirror `validW`/`validH` clipping already used for `sprcnvsarea` if the scaled sprite would overflow — in practice it won't, since a single 8x8 sprite at max zoom 8 is exactly 64x64, matching the box).

## Acceptance criteria

- [ ] Pressing 1-8 over the sprite editor assigns/clears the corresponding frame slot to/from the currently selected sprite (`sprNmbr`), reflected immediately in the frame-slot strip.
- [ ] Pressing 9 clears all 8 frame slots.
- [ ] `animZoomBtn` cycles preview zoom through 1/2/4/8 independently of the sprite canvas zoom.
- [ ] `animSpeedBtn` cycles playback speed through the defined speed steps.
- [ ] `animLoopModeBtn` cycles Pause → Forward → Reverse → PingPong → Pause, and the preview plays back accordingly, ranging only over the first..last non-empty frame.
- [ ] All new UI fits inside the empty region left of `sprcnvsarea` (`x:0-99, y:9-86`) without overlapping existing elements.
- [ ] No existing shortcuts, tools, or layout are broken.
