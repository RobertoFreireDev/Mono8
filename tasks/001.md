# Task 001: Add a 5th sprite sheet page (960 sprites total)

## Goal
Increase the sprite sheet from 768 sprites (4 pages) to 960 sprites (5 pages), and make sure the sprite editor's page selector reflects the new page count.

## Current state
- `src/core/common/Constants.cs` (`GameDataSizes`):
  - `SpriteSheetX = 256` (32 columns)
  - `SpriteSheetY = 48 * 4 = 192` (24 rows -> 4 pages of 6 rows each)
  - `TileSize = 8`
  - `SpriteSheetColumns = 32` // SpriteSheetX / TileSize
  - `SpriteSheetRows = 24` // SpriteSheetY / TileSize
  - `MaxSpriteIndex = SpriteSheetColumns * SpriteSheetRows - 1`
- `src/editor/SpriteEditor.cs`:
  - `VisibleRows = 6` (rows shown per page)
  - `SpritePages = Constants.GameDataSizes.SpriteSheetRows / VisibleRows` (currently 4)
  - `pageButtons` array and page-selector rendering already size themselves off `SpritePages`, so the selector adapts automatically once the row/column constants change.

32 columns x 24 rows = 768 sprites currently. Adding one more 6-row page = 30 rows total -> 32 x 30 = 960 sprites.

## Changes required
1. In `Constants.GameDataSizes`:
   - `SpriteSheetY = 48 * 5` (240) — add one more page of 48px height.
   - `SpriteSheetRows` comment/value updates to `30` (`SpriteSheetY / TileSize`).
   - `MaxSpriteIndex` recalculates automatically (`32 * 30 - 1 = 959`).
2. Verify `SpriteEditor.cs` page selector: `SpritePages` will become `30 / 6 = 5` automatically, and `pageButtons` will allocate 5 entries — confirm the page icon row still fits on screen (`Constants.Screen.ResolutionX - SpritePages * size`) without overlapping other UI.
3. Check any other place that assumes 4 pages or a fixed `SpriteSheetY`/`SpriteSheetRows` (e.g. spritesheet asset/texture size, serialization/save format for sprite data, export/import code) and update sizes accordingly so the extra page's data is allocated and persisted correctly.
4. Update/verify the sprite sheet texture asset dimensions if the graphics data is stored as a fixed-size image or byte array sized off the old constants.

## Acceptance criteria
- Sprite sheet holds 960 sprites (32 columns x 30 rows).
- Sprite editor shows 5 selectable pages, each with 6 rows, and switching pages works correctly (drawing, selection, copy/paste, flip/rotate, undo/redo all still operate within page bounds).
- No hardcoded reference to 4 pages, 24 rows, or 768 sprites remains.
