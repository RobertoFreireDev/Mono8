# Task 004: Animation editor and AnimSheet data system

## Goal
Add a data model (`AnimSheet`) and an in-engine editor (`AnimEditor`) for building sprite-sheet-driven animations: 4 independent draw layers, each holding 256 columns of animation data (source rect + draw offset via `sspr`), with a shared per-column frame-multiplication factor driving a global `frameindex`. The editor must support editing column/layer parameters, toggling/reordering layer visibility, copy/paste/remove of a column's anim data, hover-to-inspect frame index, and a forward/reverse/ping-pong preview loop between two frame indices. There is intentionally no named/tagged animation concept and no playback state API (no Play/Pause/Stop/IsPlaying) â€” the game or editor always supplies `frameindex` explicitly and owns all timing/looping logic itself.

## Current state
- No animation code exists anywhere in `src` today (no `Anim`, `AnimSheet`, `frameindex`, `.anims`) â€” this is a greenfield data type and editor, but must follow the exact conventions already used by the other three sheet types.
- `sspr` is the only draw primitive this feature calls into: `Mono8API.cs:141-145` (`public void sspr(int sx, int sy, int sw, int sh, int dx, int dy, int dw = -1, int dh = -1, bool flipX = false, bool flipY = false)`), which delegates to `SpriteSheet.DrawSub` (`SpriteSheet.cs:510`). Any new `animdraw`-style API must ultimately produce the same six spatial args (`sx, sy, sw, sh, dx, dy`) plus flip flags.
- Bounds to clamp against live in `Constants.cs` `GameDataSizes`: `SpriteSheetX=256`, `SpriteSheetY=192`, `TileSize=8`, `SpriteSheetColumns=32`, `SpriteSheetRows=24`, `MaxSpriteIndex`. `sx,sy,sw,sh` for every anim column must stay within `SpriteSheetX`/`SpriteSheetY`.
- File-extension + load/save convention (`Constants.cs:51-75`, `File.Extensions`): `Sfx="sfx", Music="music", SpriteSheet="gfx", MapSheet="map", Flags="gff", IconSheet="icons", Save="save"`. A new `Anims = "anims"` constant follows this pattern directly.
- `Mono8API.Load()`/`Save()` (`Mono8API.cs:18-38`) centralize all persistence: each resource type has its own `LoadX(string[])` and `ToXLines(): string[]` pair (e.g. `SpriteSheet.LoadSprites(...)`, `SpriteSheet.ToSheetLines()`, `SpriteSheet.ToFlagLines()`), called via `FileIO.Read`/`FileIO.Write(Constants.File.Name, extension, content, path)`. `Constants.File.Name = "data"`, so the new file is `data.anims`, produced/consumed the same way. The csproj's `data.*` content wildcard already picks up new files with no project-file change needed.
- Editor architecture: `src/editor/IEditor.cs` defines `Init()/Update(float)/Draw()`. `SpriteEditor` (`src/editor/SpriteEditor.cs`) is currently the *only* implementation, instantiated once and unconditionally at `Mono8API.cs:15` (`editor = new SpriteEditor(this)`), with `Mono8API.editor` a single hardcoded field (`Mono8API.cs:5`). Task 003 (shared icon-based editor-switcher bar) specs a fix for this but **has not been implemented** â€” there is no `EditorMenuBar.cs` and no multi-editor registry yet, so a new `AnimEditor` has nothing to plug into today.
- Reusable UI prior art in `SpriteEditor.cs` that the new editor should follow rather than reinvent:
  - `Button.cs`: simple icon button with `pal()` dim-when-unselected convention.
  - Grid/page selection pattern (`sprvwrarea`, `pageButtons`, `SpriteEditor.cs:39,73-78,201-211,268-275,424-435`): a scrollable/paged strip of tiles the user clicks to select an index, with a highlighted-selection outline (`SpriteEditor.cs:339-349`).
  - Keyboard-shortcut copy/paste/undo/redo/move/delete/flip/rotate pattern via `KeybrdInput` (`SpriteEditor.cs:110-179`), all operating on a "current region" computed from the current selection (`CurrentCanvasRegion()`, `SpriteEditor.cs:85-91`).
  - Zoom/scale clamping via `Math.Clamp(SprSclIdx, 0, Zooms.Length - 1)` (`SpriteEditor.cs:193`) â€” the same guard style should clamp every anim parameter.
  - Hover/selection label rendering (`sprNmbrLabelArea`, `SpriteEditor.cs:41,71,437-441`) â€” reusable pattern for the "show frameindex on hover" requirement.
- Confirmed scope decision: `fm` (frame multiplier) is **shared across all 4 layers per column** â€” there is exactly one 256-entry fm table, not one per layer. Only `sx, sy, sw, sh, offsetX, offsetY` are stored independently per (layer, column) pair. All 4 layers therefore share one global frameindexâ†’column mapping.

## Changes required

2. Create `src/core/anims/AnimSheet.cs` (mirroring `SpriteSheet`/`MapSheet`/`IconSheet` placement under `src/core/<domain>`), holding:
   - A per-(layer, column) struct/array of `{ int sx, sy, sw, sh, offsetX, offsetY }` sized `[AnimLayers][AnimColumns]`.
   - A single shared `int[AnimColumns] fm` (frame multiplier), clamped to a minimum of 1.
   - A precomputed frame-index table (recalculated whenever any `fm` entry changes) mapping: global `frameindex â†’ column index` (for drawing) and `column index â†’ starting frameindex` (for the editor's hover display). Total frame count = sum of `fm` across all 256 columns.
   - `LoadAnims(string[])` / `ToAnimLines(): string[]`, following the exact serialization convention used by `SpriteSheet.LoadSprites`/`ToSheetLines` (same file, same delimiter/line-per-record style).
   - Layer visibility state (`bool[AnimLayers]`, default all visible) and layer draw-order state (`int[AnimLayers]` permutation, default identity).
   - A draw method (e.g. `Draw(int layer, int frameIndex, int dx, int dy, bool flipX = false, bool flipY = false)`) that: clamps `layer` to `[0, AnimLayers)` and `frameIndex` to the valid computed total-frame range, resolves the column for that frameindex via the shared table, reads that layer+column's `sx,sy,sw,sh,offsetX,offsetY`, clamps `sx,sy,sw,sh` to `SpriteSheetX`/`SpriteSheetY` bounds, and calls into `SpriteSheet.DrawSub` (the same primitive `sspr` uses) with `dx+offsetX, dy+offsetY`.
   - `AnimL` behavior: a method to enable/disable one or more layers (decide exact overload â€” e.g. `AnimL(int layer, bool visible)` for a single layer, called per layer for "layer(s)") and a no-arg `AnimL()` that resets all layers to visible, mirroring the "reset" language in the spec.
   - `AnimOL` behavior: a method to set the full layer draw-order permutation (e.g. `AnimOL(int[] order)`, validated as a permutation of `0..AnimLayers-1`) and a no-arg `AnimOL()` that resets to identity order.

## Acceptance criteria
- `AnimSheet` persists correctly: editing anim data, saving, and reloading (`data.anims`) round-trips all 4 layers Ă— 256 columns of `sx,sy,sw,sh,offsetX,offsetY` plus the shared 256-entry `fm` table without loss.
- Calling the new `animdraw` API with a given layer and frameindex draws the correct sub-rect (per that layer's column data at that frameindex) at `dx+offsetX, dy+offsetY` via the same underlying `sspr`/`DrawSub` path used elsewhere, with flip flags honored.
- Changing any column's `fm` correctly recalculates the total frame count and every column's frameindex range; hovering a column in the editor shows the up-to-date frameindex range for that column.
- `animl`/`animol` correctly enable/disable and reorder layers, including their documented no-argument reset behavior, and disabled layers are skipped by draw/preview.
- In the `AnimEditor`, copy/paste correctly transfers one column's full data set to another layer/column, and remove/clear correctly resets a column to defaults â€” verified by round-tripping through the editor UI.
- The preview loop correctly plays frameindex Aâ†’B forward, reverse, and ping-pong without drawing out-of-range frames.
- No out-of-bounds sheet reads/writes are possible: every editable parameter (layer, column, `sx/sy/sw/sh`, `fm`, preview A/B, frameindex passed to `animdraw`) is clamped to valid ranges even under adversarial/edge-case input (e.g. dragging a selection past the sheet edge, setting `fm` to 0 or negative, requesting a frameindex beyond the current total).
- No regression to existing `SpriteEditor` functionality, `sspr`/`spr` drawing, or existing save/load of `data.gfx`/`data.map`/`data.gff`/`data.icons`.
- The temporary dev wiring to reach `AnimEditor` is clearly marked as temporary and does not conflict with or block task 003's future editor-switcher implementation.
